# Getting Started with Issue Hunters

> Learn how to enable Issue Hunters and configure them to monitor your repositories for issues, vulnerabilities, and code quality problems.

## Quick Setup

### 1. Enable Issue Hunters
```bash
# Enable Issue Hunters for your current repository
gammacode hunters enable

# Enable for a specific repository
gammacode hunters enable --repo username/repository-name

# Enable for your entire organization
gammacode hunters enable --org your-organization
```

### 2. Initial Configuration
```bash
# View current configuration
gammacode hunters config

# Set basic monitoring preferences
gammacode hunters config set --frequency daily --severity medium
```

### 3. First Scan
```bash
# Run your first manual scan
gammacode hunters scan

# View results
gammacode hunters list
```

## Configuration Options

### Scan Frequency
Choose how often Issue Hunters should scan your code:

```bash
# Available frequencies
gammacode hunters config set --frequency continuous  # Real-time monitoring
gammacode hunters config set --frequency hourly      # Every hour
gammacode hunters config set --frequency daily       # Once per day (default)
gammacode hunters config set --frequency weekly      # Weekly scans
```

### Issue Types
Select which types of issues to monitor:

```bash
# Enable specific issue types
gammacode hunters config set --types security,performance,quality

# Available types:
# - security: Security vulnerabilities and risks
# - performance: Performance issues and bottlenecks
# - quality: Code quality and maintainability
# - dependencies: Dependency and package issues
# - compliance: Compliance and standards violations
```

### Severity Levels
Set the minimum severity level for notifications:

```bash
# Set severity threshold
gammacode hunters config set --min-severity high

# Available levels: critical, high, medium, low
```

## Repository Setup

### Automatic Repository Detection
Issue Hunters automatically detect supported project types:

- **JavaScript/TypeScript** - Node.js, React, Vue, Angular projects
- **Python** - Django, Flask, FastAPI applications
- **Java** - Spring Boot, Maven, Gradle projects
- **Go** - Go modules and applications
- **PHP** - Laravel, Symfony, WordPress projects
- **Ruby** - Rails, Sinatra applications
- **C#/.NET** - ASP.NET, .NET Core projects

### Custom Configuration File
Create a `.gammacode/hunters.yml` file for advanced configuration:

```yaml
# .gammacode/hunters.yml
version: 1

# Global settings
settings:
  frequency: daily
  min_severity: medium
  
# Issue type configurations
issue_types:
  security:
    enabled: true
    custom_rules:
      - rule: "no-hardcoded-secrets"
        patterns: ["password.*=.*['\"]", "api_key.*=.*['\"]"]
        
  performance:
    enabled: true
    thresholds:
      memory_usage: 100MB
      execution_time: 5s
      
  quality:
    enabled: true
    exclude_patterns:
      - "*.test.js"
      - "*.spec.ts"
      - "test/**/*"

# Notification settings
notifications:
  slack:
    webhook_url: ${SLACK_WEBHOOK}
    channels:
      critical: "#alerts"
      high: "#dev-team"
      
  email:
    recipients: 
      - "team@company.com"
    frequency: daily_digest
    
# Integration settings
integrations:
  github:
    create_issues: true
    labels: ["bug", "security", "issue-hunter"]
    
  jira:
    project_key: "DEV"
    issue_type: "Bug"
```

## Team Collaboration

### Multi-Repository Setup
For organizations with multiple repositories:

```bash
# Enable for all repositories in an organization
gammacode hunters enable --org your-org --all

# Enable with specific configuration
gammacode hunters enable --org your-org --config production.yml

# Bulk configuration
gammacode hunters config set --org your-org --frequency daily --severity high
```

### Access Control
Configure team access and permissions:

```bash
# Add team members
gammacode hunters team add john@company.com --role developer
gammacode hunters team add jane@company.com --role admin

# Set up approval workflows
gammacode hunters config set --require-approval critical,high
gammacode hunters config set --approvers jane@company.com,tech-lead@company.com
```

### Dashboard Access
Team members can access the Issue Hunters dashboard at:
- **Organization Dashboard**: `https://gammacode.dev/hunters/org/your-org`
- **Personal Dashboard**: `https://gammacode.dev/hunters/personal`
- **Repository View**: `https://gammacode.dev/hunters/repo/owner/repo-name`

## Integration with CI/CD

### GitHub Actions
Add Issue Hunters to your GitHub workflow:

```yaml
# .github/workflows/issue-hunters.yml
name: Issue Hunters Scan
on: [push, pull_request]

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Issue Hunters Scan
        uses: gammacode/issue-hunters-action@v1
        with:
          api-key: ${{ secrets.GAMMACODE_API_KEY }}
          fail-on-severity: high
          comment-pr: true
```

### GitLab CI
```yaml
# .gitlab-ci.yml
issue_hunters_scan:
  stage: test
  script:
    - curl -fsSL https://gammacode.dev/install | bash
    - gammacode hunters scan --ci --fail-on high
  artifacts:
    reports:
      codequality: issue-hunters-report.json
```

### Jenkins Pipeline
```groovy
pipeline {
    agent any
    stages {
        stage('Issue Hunters Scan') {
            steps {
                sh 'gammacode hunters scan --output jenkins'
                publishHTML([
                    allowMissing: false,
                    alwaysLinkToLastBuild: true,
                    keepAll: true,
                    reportDir: 'issue-hunters-reports',
                    reportFiles: 'index.html',
                    reportName: 'Issue Hunters Report'
                ])
            }
        }
    }
}
```

## Troubleshooting Setup

### Common Issues

**Issue Hunters not detecting issues:**
```bash
# Check repository support
gammacode hunters detect

# Verify configuration
gammacode hunters config validate

# Run diagnostic
gammacode hunters diagnose
```

**Authentication problems:**
```bash
# Verify API key
gammacode auth status

# Re-authenticate if needed
gammacode login
```

**Permission issues:**
```bash
# Check repository permissions
gammacode hunters permissions check

# Request access
gammacode hunters permissions request --repo owner/repo-name
```
